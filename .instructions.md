# VVV_STOCK Project Instructions for Claude

## Project Overview
VVV_STOCK is a full-stack Stock Inventory Management Application combining:
- **Frontend**: React 18 (Vite) - localhost:5173
- **Backend**: Python Flask REST API - localhost:5000
- **Database**: MySQL (Hostinger hosting)
- **Purpose**: Real-time inventory tracking, stock operations, EOD counting, WhatsApp reporting

## Tech Stack
- React 18.2.0 with Hooks
- Vite 4.3.0 (build tool)
- Flask 2.3+ (Python backend)
- SQLAlchemy 2.0+ (ORM)
- MySQL 5.7+ (database)
- XLSX 0.18.5 (Excel export)

### Current Variable Naming (Use These)
- `CC` = Category Colors
- `CB` = Category Backgrounds
- `CATS` = Categories array
- `st()` = Status function (OUT/LOW/OK)
- `todayStr()` = Today date string
- `timeStr()` = Current time string
- `STORE_KEY` = "vvv_v4" (localStorage key)
- `load()` = Load from localStorage
- `save()` = Save to localStorage

### Component Structure
- Use inline styles (no separate CSS files)
- Use React hooks: useState, useEffect, useMemo, useRef
- Memoize expensive computations with useMemo
- Keep components functional, not class-based

### Data Structure (Products)
```javascript
{
  id: number,
  brand: string,
  item: string,        // Product name (Tamil)
  pack: string,        // Size (e.g., "50kg")
  qty: number,         // Current quantity
  cat: string          // Category
}
```

### Key Functions to Maintain
- `addStock(id, qty)` - Increase stock
- `addNewProduct(brand, item, pack, cat)` - Create product
- `deleteProduct(id)` - Delete product
- `previewEOD()` - Preview EOD snapshot
- `applyEOD()` - Save EOD count
- `exportExcel()` - Export to Excel
- `sendWhatsApp()` - Format WhatsApp message

## Common Issues & Solutions

### React app not updating
- Check localStorage key: "vvv_v4"
- Check browser cache
- Clear and refresh (Ctrl+Shift+Delete)
- Check console for errors

### Excel export failing
- Verify XLSX library installed
- Check data format (no undefined values)
- Test with sample data

### API not responding
- Check Flask is running (localhost:5000)
- Check CORS configuration
- Verify MySQL connection string
- Check firewall/network
